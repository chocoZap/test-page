要件

React UI (受講者用) 回答サンプル

🎯 完成イメージ / 制約

守るポイント

UI

  • PC 3列 / SP 1列
  • カード間に余白・影
  • ボタンのhover/activeを用意

実装

  • 関心ごと別にコンポーネント分割
  • 親→子はprops、子→親はイベントで通知
  • ステートの初期値・型を明示

品質

  • 不要な再レンダーを避ける（メモ化は任意）

✅ セルフチェック

提出前に確認

機能

  • データが正しくカードに表示される
  • フィルタ/ソートが反映される
  • 空・ローディング・エラーを出し分け

コード

  • コンポーネントが単一責務
  • propsの型・初期値が明確
  • 不要なconsole.logがない

理解

  • stateとpropsの違い
  • 再レンダーが起きるタイミング
  • キーが必要な理由
